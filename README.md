This script is a false positive analyzer for Palo Alto Networks AIRS (AI Runtime Security) scan results. Here's what it does: 

  Pre-req: Export a log file from Log Viewer for Prisma AIRS --> AI Runtime Security API
                                                                                                                                                                                                                                                    
  Purpose: Takes scan results that were flagged as "malicious" from the csv file exported from the pre-req by the AIRS API and assesses how likely each detection is a false positive.                                                                                                          
                                                                                                                                                                                                                                                    
  Workflow:                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                    
  1. Input: Reads a CSV file containing scan IDs with a "Verdict" column, filtering for entries marked "malicious."                                                                                                                                 
  2. API Calls: For each malicious scan ID, it queries two AIRS API endpoints:
    - /v1/scan/results — gets the detection flags (prompt injection, toxic content, DLP, URL filtering, malicious code, source code, contextual grounding)                                                                                          
    - /v1/scan/reports — gets detailed report data (confidence, categories, content snippets)                                                                                                                                                       
  3. False Positive Scoring: Applies a heuristic scoring system to each detection flag:                                                                                                                                                             
    - Low/moderate confidence → more likely FP                                                                                                                                                                                                      
    - Business/operational keywords in the snippet (e.g., "management", "compliance", "deployment") → more likely FP                                                                                                                                
    - Toxic content flag but no actual toxic keywords in the content → more likely FP                                                                                                                                                               
    - Prompt injection flag but content looks like a structured classification task → more likely FP                                                                                                                                                
    - Very short snippets → more likely FP                                                                                                                                                                                                          
  4. Output: Writes results to airs_malicious_violations.csv with columns for scan ID, direction (prompt/response), detection flag, confidence, FP likelihood ("Likely FP", "Possible FP", or "Review"), reasoning, and a snippet preview.          
                                                                                                                                                                                                                                                    
  Concurrency: Uses a thread pool (5 workers) with rate-limit handling (exponential backoff on 429s) for efficient API querying.
